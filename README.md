# TermRepo WebSite

TermRepo WebSite 是 `TermRepo` 项目中的指南站前端，用来对外说明产品定位、功能价值、安装方式、使用方法、常见问题与路线图。

它不是管理后台，也不是云服务控制台。这个站点更像一个面向开发者的产品说明站，核心任务是帮助用户快速理解：

- TermRepo 是什么
- 它解决什么问题
- 当前已经能做什么
- 应该如何安装和使用
- 后续会往哪里发展

## 项目定位

根据当前项目文档，TermRepo 的核心方向是：

- 面向开发者术语积累、理解、管理与复用
- 本地优先
- 轻量
- 即装即用
- 工具感强

因此，本指南站的表达也应保持一致：

- 清楚
- 克制
- 对开发者友好
- 少营销话术
- 少夸张承诺

不应把站点写成重 SaaS 官网，也不应过度强调登录、套餐、额度或尚未完成的云能力。

## 当前状态

当前目录已经完成了 Vue 3 + Vite + TypeScript 的前端脚手架初始化，但页面内容仍基本是默认模板，尚未进入正式的指南站内容开发阶段。

现阶段可以把它理解为：

- 一个已经可运行的前端站点骨架
- 一个准备承接指南站页面与内容的项目目录
- 一个后续会逐步替换默认示例组件的真实业务前端

## 建议承载的内容

这个站点后续适合承载以下页面或内容模块：

- 首页
- 功能介绍页
- 使用指南页
- 常见问题页
- 更新日志页

建议重点覆盖的信息包括：

- 项目定位
- 适合什么人使用
- 典型使用场景
- 与普通词典工具的区别
- VS Code 插件安装方式
- 词条收藏、搜索替换、导入导出等核心流程
- 当前已实现能力与后续路线图

## 不适合承载的内容

以下内容不应作为本指南站的主体职责：

- 重管理功能
- token 管理
- 套餐额度管理
- 用户服务台
- 内测资格发放

这些内容更适合未来独立的管理站或云服务站处理。

## 目录结构

```text
termrepo-guide-site/
├─ public/                 静态资源
├─ src/
│  ├─ assets/              样式与素材
│  ├─ components/          组件
│  ├─ router/              路由配置
│  ├─ views/               页面级视图
│  ├─ App.vue              应用入口组件
│  └─ main.ts              应用启动入口
├─ index.html
├─ package.json
├─ tsconfig.json
└─ vite.config.ts
```

当前 `src/` 中仍保留 Vue 默认示例组件与示例页面，后续应逐步替换为真正的指南站页面结构。

## 建议的后续开发方向

可以按下面顺序推进：

1. 清理默认脚手架页面与示例组件
2. 搭建指南站基础布局与导航
3. 先完成首页、功能介绍页、使用指南页
4. 补充 FAQ 与路线图内容
5. 再根据需要接入更新日志、下载入口或外链跳转

## 文案与页面边界

在编写页面文案时，应优先强调：

- 开发者术语积累与复用
- 本地优先
- 轻量
- 当前已经落地的功能

应谨慎表达或避免过度突出：

- 重商业化
- 强登录
- 尚未实现的完整云服务能力

如果要提及 AI、同步或后续网页端能力，建议明确它们属于未来增强项，而不是当前插件可用性的前提。
